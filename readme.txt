Purpose
-------
Customer-facing documentation and knowledge work: BRDs, design notes, FAQ, HelpJuice-style help content, raw inputs, and graphify outputs for search and synthesis.

How to use
----------
1. Browse BRDs and Design for authoritative requirements; use FAQ and HelpJuice for end-user language.
2. Point documentation or AI tools at raw/ for unprocessed source, and graphify-out/ for structured graph exports from analysis pipelines.
3. Large .zip archives are snapshots; prefer editing extracted copies and re-zipping when sharing.
4. Keep personally identifiable information out of shared copies where possible.
